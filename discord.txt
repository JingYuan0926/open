You're constantly checking yields and rebalancing. Existing bots help but they're either centralized servers that can be shut down or black boxes where you can't verify what they actually decided.

Zwarm is a network of three specialist agents that collaborate to manage portfolios autonomously. There's an executor, a seeker, and a critic, each running as its own separate AXL node. AXL is the entire communication backbone.

The executor monitors positions and makes rebalancing decisions using TEE-verified inference on 0G Compute, storing every decision with proofs on 0G Storage. Once it builds a track record, it broadcasts strategy availability across the mesh using GossipSub.

The seeker listens for those broadcasts. When it finds a strategy matching its risk profile, it queries the publisher over A2A for the track record, verifies the proofs on 0G Storage, and subscribes through KeeperHub's marketplace where the logic stays private. Payments flow agent-to-agent via x402.

The critic uses Convergecast to aggregate performance data across the network and publishes ratings back to the mesh. It just observes and rates.

So I'm using AXL across four protocols- Send/Recv, A2A, MCP routing, Convergecast with three separate nodes communicating in real time. The whole point is showing an agent economy can exist without a central coordinator.

GenGuard is a decentralized agent firewall for AI agents built around Gensyn AXL as the core coordination network, with 0G providing verifiable storage and compute.

 It addresses real-world failures like the Meta AI Agent Data Exposure, where an agent acted beyond its scope and exposed sensitive data due to lack of validation. More broadly, industry reports show autonomous agents are involved in a growing share of AI breaches, while visibility into their actions remains limited. 

When an agent attempts a sensitive action, it creates an intent payload and broadcasts it over the AXL peer-to-peer mesh using A2A messaging. Independent guardian agents on separate AXL nodes receive the intent, evaluate it, and return approve/block decisions. 

All coordination, communication, and consensus happen fully peer-to-peer within AXL, with no central server. Unlike centralized safety tools—which introduce single points of failure, control, and misconfiguration—GenGuard enforces validation through distributed consensus. 

Policies are fetched from 0G KV Store, reasoning runs via 0G Compute (TEE-backed), and all decisions are stored on 0G Storage with hashes anchored on 0G Chain for immutable auditability. 

The result is a trustless governance layer where AI agents cannot perform sensitive actions without decentralized peer validation.

1) covnergecast is one way to perform a gather.  In the /topology endpoint you have access to the network topology, so if you need to use ephemeral nodes you can orchestrate direct send/recv to the nodes that are static. Project: Huddle — Intent-Matched Bulk Purchase Coalitions

The Pain:
DeFi solved coordination for capital — AMMs let strangers pool liquidity without trusting each other. Nobody solved it for purchasing power. Any central broker aggregating purchase intent becomes a front-running machine — selling intent to the retailer or taking a rake. Same order-flow problem Flashbots solved in DeFi. Nobody solved it for consumer purchasing.

AXL fixes this at infrastructure level. Buyer agents broadcast hashed intents — H(product_id  max_price

 deadline || nonce) — over AXL's encrypted P2P mesh. No node sees cleartext. When ≥10 matching hashes are detected, agents reveal intents only to each other over direct A2A channels. gm friends  As part of my project, I am working on a quick&dirty TS SDK for AXL and I just published two small TypeScript packages for working with Gensyn AXL:

@0xagentio/axl-client - thin TS client for the AXL node HTTP bridge,
@0xagentio/axl-local - local dev/test wrapper to spin up and manage the AXL node processes from TypeScript (I prefer programmable processes x)),

This is still pretty basic but I thought maybe this can make someone elses life a little easier :) 

 Note: this does not reimplement AXL, you still need the real Gensyn AXL node binary and provide it to the SDK 

Install:

  
npm install @0xagentio/axl-client @0xagentio/axl-local

# then build the AXL node

git clone https://github.com/gensyn-ai/axl
cd axl
go build -o node ./cmd/node/


Then use it from TS:


import { startLocalAxlNetwork } from '@0xagentio/axl-local';

const network = await startLocalAxlNetwork({
  binaryPath: '/Users/trivo/Documents/GitHub/axl/node',
  workingDirectory: '.axl',
  nodes: [
    { name: 'alice', apiPort: 9101, listen: ['tls://127.0.0.1:9201'] },
    { name: 'bob', apiPort: 9102, peers: ['tls://127.0.0.1:9201'] },
  ],
});

const alice = network.node('alice');
const bob = network.node('bob');

await alice.client.send({
  peerId: bob.peerId,
  body: new TextEncoder().encode('hello bob'),
});

const msg = await bob.client.recv();
if (msg) {
  console.log(new TextDecoder().decode(msg.body));
} else {
  console.log('no message received');
}

await network.stop();

Useful if you want to test local P2P agent communication with AXL from TypeScript and maybe even spin up multiple ones  (edited)Tuesday, 28 April 2026 at 15:33
